Mutate an Array Declared with const

Objects that are declared with const are still able to be mutated, but they cannot be reasigned. Trying to re-declare the const (e.g myArray = [1, 2, 3]) will result in an error, but doing pop(), shift() or a specific change to a part of the structure like "myArray[2] = 45" will not result in an error.

Only the name and the type of the structure are not mutable when using const.

Prevent Object Mutation

The function Object.freeze() will prevent any changes to the object passed as a parameter to the function from taking place and will result in an error.

You would obviously need to use parameters when using arrow functions.

Set Default Parameters for Your Functions

It's a nice touch to give functions default parameters, you can do so with
const userAge = (age = "not specified") => "The age is " + age;
This will either return the age of the user or the text "The age is not specified".

Use Destructuring Assignment to Assign Variables from Nested Objects

This one was kinda hard for me at first, but once it clicked it felt good.
Basically you can use destructuring in an object that is itself inside another object. Let's suppouse that we have an object called VEHICLES that has fields that are objects themselves, these fields are going to be cars and bikes which have fields for the models of the cars and bikes that they have in store, it would look something like this.

            
                const VEHICLES = {
                     cars: { ford: $150K, mitsubishi: $200K },
                     bikes: { honda: [$30K, $25K, $16K], mitsubishi: $23K }    
                 };

Now we assign them as taught:
const { cars: { ford: fordCar, mitsubishi: mitsuCar } } = VEHICLES;

The variables fordCar and mitsuCar have been created and both have the respective prices... but how do we access one of the elements in the array of the honda bikes? That's a question I have till this very day.

Use Destructuring Assignment to Assign Variables from Arrays

We can also destructure elements from a list, and differently from using the Spread Operator(...arr1), which doesn't allow us to name the variables, d they are spread inside an array, when we use the destructuring method we can do exactly that! :) 

            
                 const [ theFirst,, theThird ] = [1, 2, 3];

console.log()ing will show that theFirst and his brother theThird will have their corresponding values: 1 and 3, while the second one will be omitted since we used commas to avoid one value. Neat!

Destructuring via rest elements

Suppouse we have an array and we want to create some variables to hold specific values like the first and the third one in the last example, but we also want to store the rest of the array in a new array with a new name.
We can do that easily.

const [ a, b, ...arr ] = [1, 2, 3, 4, 5, 6];

Logging will show that a = 1, b = 2, and arr = [3, 4, 5, 6].. but something is off... right! We wanted the first and the third elements.

const [ a,, b, ...arr ] = [1, 2, 3, 4, 5, 6]; should do the trick.

Use Destructuring Assignment to Pass an Object as a Function's Parameters

We can use Destructuring Assignment inside the parameters of a function to automatically declare variables from the passed object to use inside the function. Slick and elegant!

const car_parts = ({engine, battery, tires, lights}) => { }

We are not naming the variable that is gonna hold the object and then declaring variables for the fields we want from the object inside the function, we are rather declaring the variables for the object's fields already in the parameters!

Create Strings using Template Literals

This is a feature that I thought was really cool when I was writing code in Rust. And that I found also to be in Python.
In JS they made it a bit harder to use because you need to use backsticks and not quotes (Yes, I think it makes it harder, I'm not the brightest tool in the shed mmmmokay?). Template Literals allow us the use variables and object fields inside strings.
The formula goes like this:

                
                const Bobbie = {
                    age: 16,
                    gamer: True,    
                    eyes: "Brown",
                }
                
                const eyes = `Bobbie has ${Bobbie.eyes.toLowerCase()} eyes`;
                console.log(eyes);

Write Concise Object Literal Declarations Using Object Property Shorthand

Whenever we can save time, we should, as long as the code remains readable. When defining object literals, as long as the parameters passed have the same name as the object fields that need to be assigned, we can avoid repetition and make things easier, so instead of doing return { x: x, y: y };, simply do return { x, y };.
